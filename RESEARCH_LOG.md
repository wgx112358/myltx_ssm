# Research Log

This file is the default place to record research trajectory updates, effective insights, route changes, hypotheses to verify, key experiment outcomes, and rollback points.

## Purpose

- Keep the current research path inspectable.
- Make route changes, decisions, and failures traceable.
- Support rollback after failed experiments.
- Preserve effective insights so iteration can continue from the latest solid point instead of restarting blindly.

## When To Record

Create a new entry whenever any of the following happens:

- A new effective insight appears.
- The current research route, problem judgment, or technical direction changes.
- A new hypothesis, idea, or experiment direction needs verification.
- A key experiment result is strong enough to affect the next decision.
- A rollback, abandonment, or replacement of a previous plan is needed.

## Entry Rules

- Append new entries to the end of this file in chronological order.
- Keep each entry concise but decision-useful.
- Every important route switch, experiment push, and rollback decision should be traceable to at least one entry here.
- If an experiment fails, review the latest valid entry first and continue from its rollback point.

- `Summary`: The core insight, updated judgment, new hypothesis, or result.
- `Trigger / Evidence`: Why this entry exists now, such as observations, papers, failed runs, logs, or experiment outputs.

- `Rollback Point`: The last stable assumption, version, result, or route to return to if this path fails.
